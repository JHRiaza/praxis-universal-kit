# Historical Baseline Reconstruction Protocol

### Purpose
This protocol defines how to reconstruct baseline evidence when a participant began using AI-assisted workflows before installing PRAXIS Universal Kit instrumentation.

Backfilled data is useful for contextual analysis, but it is weaker than prospectively collected data. It must be labeled, separated from live metrics where possible, and treated as reconstructed evidence rather than native telemetry.

### Scope
Use this protocol when:
- The original observation period was not instrumented by the kit.
- The researcher needs a baseline comparison before structured observation began.
- Retrospective evidence can provide useful context for interpreting live data.

Do not use this protocol to overwrite prospectively collected data.
